Vim简介，以及我为什么要学习vim

Vim简介

根据维基百科，Vim 是从vi发展出来的一个文本编辑器。Vim的第一个版本是由布莱姆·米勒在1991年发布，最初的简称是 Vi IMitation，随着功能的不断增加，正式名称改成了 Vi IMproved。

Vim的学习曲线比较陡峭，开始学习的时候比较缓慢，但是一旦掌握了基础之后，能大幅度提高编辑效率。

维基百科介绍vim有6种基本模式和5种派生模式。

基本模式：

  1. 普通模式
  2. 插入模式
  3. 可视模式
  4. 选择模式
  5. 命令行模式
  6. Ex模式

派生模式：

  1. 操作符等待模式
  2. 插入普通模式
  3. 插入可视模式
  4. 插入选择模式
  5. 替换模式

我为什么学习vim

当然不是为了装X!但也不能说完全没有。

由于有两台linux云主机，很多时候需要在线修改一些程序脚本文件，编辑一些配置文件，所以需要用到vim。（不太可能在云主机上安装desktop，实在太浪费。）

维基百科和简书上的那篇教程对于vim的一些模式的名称不统一，例如刚进入vim的默认模式维基百科称之为普通模式，而简书的教程称之为编辑模式。而且简书的教程介绍vim有三种操作模式：编辑模式、插入模式和命令模式。

就我个人来说，其实主要模式就是三种，就如简书教程来说。当然，也会用到可视模式等功能，但是可视模式就比较像普通模式（编辑模式）的一个附属功能。我也不准备深入探讨，做个vim学霸，还是实用至上。就按照简书上的介绍来学习好了。（其实也看到过老外介绍时说到有 2种主要模式～insert mode and command mode）

编辑模式/普通模式/command mode

这就是打开vim时的默认模式，在这个模式下可以通过一系列操作移动光标，剪切，黏贴，删除，插入。

这个时候最显著也是和我们平常编辑器最大的不同是：你输入的键盘字母都会被当做命令执行，而不是所见即所得的文本。

模式切换

command mode可以说是vim的核心模式，它帮助在各个模式之间穿梭。

[Insert Mode] –> [Command Mode]: Esc按键
[Command Mode] –> [Insert Mode]: 四个按键: i,a,I,A。
[Command Mode] –> [Visual Mode]: 三个按键: v, V, Ctrl+v
[Visual Mode] –> [Command Mode]: Esc按键

To switch from the command mode to the insert mode type one of

  • i …switch to insert mode before the current position（最常用）
  • a …switch to insert mode after the current position (append)（和上一个差不多，一个在前，一个在后）
  • I …jump to the first non-blank character in the current line and switch to the insert mode（行开头插入）
  • A …jump to the last character of the current line and switch to the insert mode（行末插入）

To switch from the command mode to the visual mode type one of

  • v …switch to the visual mode (character oriented)（选择字符）
  • V …switch to the visual mode (line oriented)（多行选择）
  • Ctrl-v …switch to the block-visual mode (select rectangles of text)（矩阵选择，这个最好用，可以用于多行注释取消注释）

跳转/Movement

简单的移动模式对于刚接触vim（比如说我）来说，简直反人类：

  • h …move left
  • l …move right
  • k …move up
  • j …move down

进阶移动：
老外有张图挺形象的，没好意思盗人家的图：）

  • 0 …first column of the line
  • ^ …first non-blank character of the line
  • w …jump to next word
  • W …jump to next word, ignore punctuation
  • e …jump to word-end
  • E …jump to word-end, ignore punctuation
  • b …jump to word-beginning
  • B …jump to word-beginning, ignore punctuation
  • ge …jump to previous word-ending
  • gE …jump to previous word-ending, ignore punctuation
  • g_ …jump to last non-blank character of the line
  • $ …jump to the last character of the line

简书的说明也很赞：

  • H 将光标移到屏幕上的起始行（或最上行）；
  • M 将光标移到屏幕中间；
  • L 将光标移到屏幕最后一行。

需要注意字母的大小写。H 和 L 命令还可以加数字。如 2H 表示将光标移到屏幕的第２行，3L 表示将光标移到屏幕的倒数第3行。
当将光标移到所要的行是，行内移动 光标可以用下面的命令来实现：

  • w 右移光标到下一个字的开头；
  • e 右移光标到一个字的末尾；
  • b 左移光标到前一个字的开头；
  • 0 数字０，左移光标到本行的开始；
  • $ 右移光标，到本行的末尾；
  • ^ 移动光标，到本行的第一个非空字符。

接下来是翻页：

  • Ctrl+f 在文件中前移一页（相当于 page down）；
  • Ctrl+b 在文件中后移一页（相当于 page up）；
  • Ctrl-e 一行上移
  • Ctrl-d 文件中上移1/2页
  • Ctrl-y 一行下移
  • Ctrl-u 文件中下移1/2页

还有更加高级的:

  • * 当光标停留在一个单词上，* 键会在文件内搜索该单词，并跳转到下一处；
  • # 当光标停留在一个单词上，# 在文件内搜索该单词，并跳转到上一处；
  • (/) 移动到 前/后 句 的开始；
  • {/} 跳转到 当前/下一个 段落 的开始。
  • g_ 到本行最后一个不是 blank 字符的位置。
  • fa 到下一个为 a 的字符处，你也可以fs到下一个为s的字符。
  • t, 到逗号前的第一个字符。逗号可以变成其它字符。
  • 3fa 在当前行查找第三个出现的 a。
  • F/T 和 f 和 t 一样，只不过是相反方向;
  • gg 将光标定位到文件第一行起始位置；
  • G 将光标定位到文件最后一行起始位置；
  • NG或Ngg 将光标定位到第 N 行的起始位置。
